云器 Lakehouse 数据流水线设计大全

基于官方产品文档整理 · 端到端数据管道设计参考

1. 流水线总览与选型决策树

展示所有数据流水线类型的全景图和选型决策树。根据实时性要求、数据源类型、同步范围、是否需要 CDC 等维度，选择最合适的数据管道方案。

2. 实时数据加载架构

详细展示所有实时数据加载方式：Pipe 持续导入、Studio 数据集成、SDK Streaming API、SQL INSERT/COPY INTO。每种方式适用于不同的数据源和实时性要求。

3. Kafka 数据接入全景

Kafka 接入的三种路径对比：READ_KAFKA Pipe（推荐）、Kafka 外部表 + Table Stream + Pipe、Studio 实时同步。根据场景选择最合适的接入方式。

4. 对象存储数据接入全景

OSS/S3/COS 数据接入的完整路径：OSS Pipe LIST_PURGE（通用）、OSS Pipe EVENT_NOTIFICATION（低延迟）、Volume Stream（文件变更监听）、COPY INTO（一次性批量导入）、INSERT INTO FROM VOLUME。所有路径均需先创建 Storage Connection 和 External Volume。

5. 数据库 CDC 同步全景

MySQL/PostgreSQL 等数据库的 CDC 同步方案：Studio 多表实时同步（整库镜像/多表镜像/分库分表合并）、Studio 单表实时同步、Studio 离线同步。包含数据库前置配置要求和运维 SOP。

6. 增量计算引擎详解（Dynamic Table）

Dynamic Table 的工作原理和最佳实践。MetaService 记录每张表的历史数据版本，Dynamic Table 读取 Base Table 的 Delta（变化数据），CBO 智能决策全量计算 vs 增量计算。建议使用 GP 集群（刷新时自动执行小文件合并）。

7. 变化数据捕获（Table Stream / Volume Stream）

Table Stream 记录表的 DML 变更（INSERT/UPDATE/DELETE），是构建 CDC 管道的核心对象。Volume Stream 监听外部对象存储的文件变更事件。两者都通过 DML 消费自动推进 offset，支持与 Dynamic Table 或 SQL 任务配合实现增量 ETL。

8. 物化视图详解（Materialized View）

物化视图预计算并存储查询结果，适合固定维度聚合加速场景。支持定时自动刷新和手动刷新，支持查询改写（Preview）。基表变更后物化视图自动失效，无法用于查询改写直到下次刷新。

9. 多层 ETL 架构模式（Medallion / ODS-DWD-DWS）

云器 Lakehouse 支持 Medallion Architecture（Bronze/Silver/Gold）和国内数仓分层（ODS/DWD/DWS）两种命名惯例。通过 Pipe + Dynamic Table 级联构建端到端实时 ETL 管道，各层独立设置刷新频率。典型场景包括 SCD 缓慢变化维处理。

10. 调度与运维体系

任务调度、监控告警、数据质量的完整体系。Studio 提供可视化的任务编排、DAG 依赖管理、多种告警通知渠道。支持 DDL 定义调度、Studio 调度、流式 SQL、第三方调度引擎等多种调度方式。

11. 性能调优与最佳实践

数据流水线的性能优化建议：计算资源选型、Pipe 调优、Dynamic Table 调优、小文件优化、Result Cache、文件大小建议、监控指标。
