ClickZetta Lakehouse Data Pipeline Design Guide

Based on Official Documentation · End-to-End Data Pipeline Design Reference

1. Pipeline Overview & Selection Decision Tree

A panoramic view of all data pipeline types and a selection decision tree. Choose the most suitable pipeline based on latency requirements, data source type, sync scope, and whether CDC is needed.

2. Realtime Data Loading Architecture

Detailed view of all realtime data loading methods: Pipe Continuous Consumption, Studio Data Integration, SDK Streaming API, SQL INSERT/COPY INTO. Each method is suited for different data sources and latency requirements.

3. Kafka Ingestion Full Picture

Three Kafka ingestion paths compared: READ_KAFKA Pipe (Recommended), Kafka External Table + Table Stream + Pipe, Studio Realtime Sync. Choose the best approach based on your scenario.

4. Object Storage Ingestion Full Picture

Complete paths for OSS/S3/COS data ingestion: OSS Pipe LIST_PURGE (General), OSS Pipe EVENT_NOTIFICATION (Low Latency), Volume Stream (File Change Capture), COPY INTO (One-time Bulk Import), INSERT INTO FROM VOLUME. All paths require creating a Storage Connection and External Volume first.

5. Database CDC Sync Full Picture

CDC sync solutions for MySQL/PostgreSQL and other databases: Studio Multi-table Realtime Sync (Whole DB Mirror / Multi-table Mirror / Shard Merge), Studio Single-table Realtime Sync, Studio Batch Sync. Includes database prerequisite configurations and operations SOP.

6. Incremental Compute Engine Deep Dive (Dynamic Table)

How Dynamic Table works and best practices. MetaService records historical data versions for each table. Dynamic Table reads Delta (changed data) from the Base Table, and CBO intelligently decides between full refresh vs incremental refresh. GP Cluster recommended (auto small file compaction during refresh).

7. Change Data Capture (Table Stream / Volume Stream)

Table Stream records DML changes (INSERT/UPDATE/DELETE) on a table and is the core object for building CDC pipelines. Volume Stream monitors file change events on external object storage. Both advance offset via DML consumption and can work with Dynamic Table or SQL tasks for incremental ETL.

8. Materialized View Deep Dive (Materialized View)

Materialized Views pre-compute and store query results, suited for fixed-dimension aggregation acceleration. Supports scheduled auto-refresh and manual refresh, and Query Rewrite (Preview). After base table changes, the MV is automatically invalidated and cannot be used for query rewrite until the next refresh.

9. Multi-Layer ETL Architecture Patterns (Medallion / ODS-DWD-DWS)

ClickZetta Lakehouse supports both Medallion Architecture (Bronze/Silver/Gold) and domestic DW layering (ODS/DWD/DWS) naming conventions. Build end-to-end realtime ETL pipelines via Pipe + Dynamic Table cascading, with independent refresh frequencies per layer. Typical scenarios include Slowly Changing Dimensions (SCD) processing.

10. Scheduling & Operations System

Complete system for task scheduling, monitoring & alerting, and data quality. Studio provides visual task orchestration, DAG dependency management, and multiple alerting notification channels. Supports DDL-defined scheduling, Studio scheduling, Streaming SQL, and third-party scheduling engines.

11. Performance Tuning & Best Practices

Performance optimization recommendations for data pipelines: compute resource selection, Pipe tuning, Dynamic Table tuning, small file optimization, Result Cache, file size recommendations, and monitoring metrics.
