云器 Lakehouse 架构总览

基于官方产品文档整理 · 全托管湖仓一体平台

1. 整体平台架构

云器 Lakehouse 采用存算分离、Serverless 弹性架构，底层基于开放存储格式（Iceberg / Parquet / ORC），支持 AI 增强分析。多云部署（阿里云 / 腾讯云 / AWS）。

2. 对象层级模型

云器 Lakehouse 的对象层级：账户 → 服务实例 → 工作空间 → {Schema, Share, Connection, External Catalog}。Schema 下包含表、视图、函数、Volume、Stream、Pipe、索引、同义词等数据对象。

3. 数据导入架构

云器 Lakehouse 支持多种数据导入路径：批量文件导入（COPY INTO）、流式 Pipe 持续导入（Kafka / OSS）、Studio 可视化同步任务（离线/实时/CDC）、以及 SDK 程序化写入。Pipe 运行在通用型（GP）集群上。

4. 数据处理流水线（增量计算）

云器 Lakehouse 提供 Pipe（持续摄入）、Table Stream（CDC 变更捕获）、Dynamic Table（声明式增量刷新）和 Materialized View（定时全量刷新）四种增量处理机制，可级联构建多层 ETL 管道。

5. 安全架构

云器 Lakehouse 采用多层安全体系：身份认证（SSO/MFA）、RBAC 权限控制、列级动态脱敏、网络策略（IP 白名单/黑名单，Deny 优先）、PrivateLink 私网连接、以及传输/存储加密。

6. 连接与生态集成

云器 Lakehouse 提供丰富的连接方式，兼容主流 BI 工具、AI 框架、数据集成平台和大数据生态。支持 JDBC、SQLAlchemy、MySQL 协议、MCP Server 等多种接入方式。

7. AI 增强分析能力

云器 Lakehouse 内置 AI Gateway（模型管理与治理）、AI 函数（AI_COMPLETE / AI_EMBEDDING）、向量索引（HNSW）、External Function（调用自定义模型）、DataGPT（自然语言查询）以及 MCP Server（AI Agent 接入）。

8. 存储层详细架构

存储层基于开放格式（Apache Iceberg），支持 Time Travel、ACID 事务、多种索引类型，以及 External Catalog 联邦查询。VECTOR(FLOAT, n) 支持最大 65535 维向量。

9. 数据类型体系

云器 Lakehouse 支持完整的 SQL 数据类型，包括基础类型、复杂类型（ARRAY/MAP/STRUCT）、AI 专用的 VECTOR(FLOAT/TINYINT, n) 类型，以及高效的 BITMAP 类型。

10. 关键 SQL 语法速查

常用 DDL/DML 语法参考，涵盖建表、数据导入、增量处理、AI 函数等核心操作。注意：普通表不支持 CREATE OR REPLACE，Dynamic Table / View / MV 支持。

11. Studio 任务调度与运维体系

Lakehouse Studio 提供完整的任务开发、调度、运维和监控能力。支持 SQL / Python / Shell / JDBC 任务类型，任务组 DAG 编排，数据质量规则，以及监控告警（飞书/企业微信 webhook）。
