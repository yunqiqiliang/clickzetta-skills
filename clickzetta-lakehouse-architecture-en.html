ClickZetta Lakehouse Architecture Overview

Based on Official Product Documentation · Fully Managed Lakehouse Platform

1. Overall Platform Architecture

ClickZetta Lakehouse adopts a compute-storage separation, Serverless elastic architecture, built on open storage formats (Iceberg / Parquet / ORC), with AI-enhanced analytics. Multi-cloud deployment (Alibaba Cloud / Tencent Cloud / AWS).

2. Object Hierarchy Model

ClickZetta Lakehouse object hierarchy: Account → Instance → Workspace → {Schema, Share, Connection, External Catalog}. Schema contains data objects such as tables, views, functions, Volumes, Streams, Pipes, indexes, and synonyms.

3. Data Ingestion Architecture

ClickZetta Lakehouse supports multiple data ingestion paths: bulk file import (COPY INTO), streaming Pipe continuous ingestion (Kafka / OSS), Studio visual sync tasks (Batch/Realtime/CDC), and SDK programmatic writes. Pipes run on General Purpose (GP) clusters.

4. Data Processing Pipeline (Incremental Computation)

ClickZetta Lakehouse provides four incremental processing mechanisms: Pipe (continuous ingestion), Table Stream (CDC change capture), Dynamic Table (declarative incremental refresh), and Materialized View (scheduled full refresh), which can be cascaded to build multi-layer ETL pipelines.

5. Security Architecture

ClickZetta Lakehouse employs a multi-layer security system: identity authentication (SSO/MFA), RBAC access control, column-level dynamic data masking, network policy (IP allowlist/denylist, Deny-first), PrivateLink private connectivity, and transport/storage encryption.

6. Connectivity & Ecosystem Integration

ClickZetta Lakehouse provides rich connectivity options, compatible with mainstream BI tools, AI frameworks, data integration platforms, and big data ecosystems. Supports JDBC, SQLAlchemy, MySQL protocol, MCP Server, and more.

7. AI-Enhanced Analytics Capabilities

ClickZetta Lakehouse includes built-in AI Gateway (model management and governance), AI functions (AI_COMPLETE / AI_EMBEDDING), vector index (HNSW), External Function (custom model invocation), DataGPT (natural language querying), and MCP Server (AI Agent integration).

8. Storage Layer Detail

The storage layer is based on open formats (Apache Iceberg), supporting Time Travel, ACID transactions, multiple index types, and External Catalog federated queries. VECTOR(FLOAT, n) supports up to 65535-dimensional vectors.

9. Data Type System

ClickZetta Lakehouse supports a complete SQL data type system, including basic types, complex types (ARRAY/MAP/STRUCT), AI-specific VECTOR(FLOAT/TINYINT, n) types, and efficient BITMAP types.

10. Key SQL Syntax Quick Reference

Common DDL/DML syntax reference covering table creation, data import, incremental processing, and AI functions. Note: regular tables do not support CREATE OR REPLACE; Dynamic Table / View / MV do.

11. Studio Task Scheduling & Operations

Lakehouse Studio provides comprehensive task development, scheduling, operations, and monitoring capabilities. Supports SQL / Python / Shell / JDBC task types, task group DAG orchestration, data quality rules, and monitoring & alerting (Feishu/WeCom webhook).
