以下是创建物化视图 dws.mv_daily_revenue 的 SQL 语句，每小时自动刷新，统计每天每个 region 的 revenue 总和，数据来自 dwd.orders_clean。

-- 创建物化视图
CREATE MATERIALIZED VIEW dws.mv_daily_revenue
REFRESH EVERY 1 HOUR
AS
SELECT
    DATE(order_time) AS order_date,
    region,
    SUM(revenue) AS total_revenue
FROM dwd.orders_clean
GROUP BY DATE(order_time), region;

说明：
1. CREATE MATERIALIZED VIEW dws.mv_daily_revenue：在 dws schema 下创建名为 mv_daily_revenue 的物化视图。
2. REFRESH EVERY 1 HOUR：设置每小时自动刷新一次物化视图数据。
3. DATE(order_time)：将订单时间截取为日期，用于按天统计（字段名根据实际表结构调整，常见字段名如 order_date、created_at、order_time 等）。
4. SUM(revenue)：对 revenue 字段求和，统计每天每个 region 的总收入。
5. GROUP BY DATE(order_time), region：按日期和地区分组聚合。

注意事项：
- 请根据 dwd.orders_clean 表的实际字段名调整 order_time 和 revenue 字段名。
- 不同数据库/数仓平台对物化视图自动刷新的语法略有差异：
  - ClickZetta / Zettapark：使用 REFRESH EVERY 1 HOUR
  - Snowflake：使用 WAREHOUSE = <warehouse_name> 并配合 Task 实现定时刷新
  - PostgreSQL：需借助 pg_cron 等扩展实现定时刷新
  - Oracle：使用 REFRESH FAST START WITH SYSDATE NEXT SYSDATE + 1/24
- 如果平台不支持 REFRESH EVERY 语法，可改用调度任务（如 Airflow、DolphinScheduler）定期执行 REFRESH MATERIALIZED VIEW dws.mv_daily_revenue;
