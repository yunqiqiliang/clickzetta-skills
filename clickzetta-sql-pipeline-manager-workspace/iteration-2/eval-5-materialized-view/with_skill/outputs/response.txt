## 创建物化视图 dws.mv_daily_revenue

根据你的需求，这是一个固定聚合场景（每天每个 region 的 revenue 总和），不需要秒/分钟级增量刷新，适合使用**物化视图（Materialized View）**。

---

### 步骤 1：创建物化视图

```sql
CREATE OR REPLACE MATERIALIZED VIEW dws.mv_daily_revenue
  COMMENT '每日各 region 收入汇总'
  REFRESH AUTO EVERY '1 hours'
  VCLUSTER = default_ap
AS
SELECT
  DATE(created_at) AS day,
  region,
  SUM(revenue) AS total_revenue
FROM dwd.orders_clean
GROUP BY 1, 2;
```

**关键语法说明：**
- `REFRESH AUTO EVERY '1 hours'`：ClickZetta 特有写法，必须带 `AUTO` 关键字，时间值必须加引号。错误写法是 `REFRESH EVERY 1 HOUR`（无引号、无 AUTO）。
- `VCLUSTER = default_ap`：指定执行刷新的计算集群，未指定时默认使用 `default_ap`。
- 物化视图是全量刷新，每小时重新计算全部数据。

---

### 步骤 2：手动触发首次刷新（可选）

创建后如需立即刷新，单独执行：

```sql
REFRESH MATERIALIZED VIEW dws.mv_daily_revenue;
```

---

### 步骤 3：验证

```sql
-- 确认物化视图已创建
SHOW TABLES WHERE is_materialized_view = true;

-- 查看结构
DESC MATERIALIZED VIEW dws.mv_daily_revenue;

-- 查询数据验证
SELECT * FROM dws.mv_daily_revenue LIMIT 10;
```

---

### 注意事项

1. **字段名确认**：上述 SQL 假设 `dwd.orders_clean` 中存在 `created_at`、`region`、`revenue` 字段。如果实际字段名不同（例如金额字段叫 `amount`），请相应调整：
   ```sql
   SUM(amount) AS total_revenue
   ```

2. **暂停/恢复自动刷新**：
   ```sql
   -- 暂停
   ALTER MATERIALIZED VIEW dws.mv_daily_revenue SUSPEND;
   -- 恢复
   ALTER MATERIALIZED VIEW dws.mv_daily_revenue RESUME;
   ```

3. **修改注释**（使用 ALTER TABLE，不是 ALTER MATERIALIZED VIEW）：
   ```sql
   ALTER TABLE dws.mv_daily_revenue SET COMMENT '每日各 region 收入汇总（更新说明）';
   ```

4. **删除**：
   ```sql
   DROP MATERIALIZED VIEW IF EXISTS dws.mv_daily_revenue;
   ```
