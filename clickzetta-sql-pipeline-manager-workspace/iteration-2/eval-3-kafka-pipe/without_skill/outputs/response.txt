要从 Kafka 持续导入订单数据到 ods.orders 表，可以使用 ClickZetta（云器科技）的 PIPE 功能来创建一个持续的 Kafka 数据导入管道。

以下是完整的 SQL 实现方案：

---

## 第一步：确认目标表存在

首先确保 ods.orders 表已创建，如果不存在需要先建表：

```sql
-- 示例建表（根据实际 JSON 字段调整）
CREATE TABLE IF NOT EXISTS ods.orders (
    order_id     STRING,
    user_id      STRING,
    product_id   STRING,
    quantity     INT,
    amount       DECIMAL(18, 2),
    status       STRING,
    created_at   TIMESTAMP,
    updated_at   TIMESTAMP
)
USING DELTA;
```

---

## 第二步：创建 Kafka PIPE

使用 CREATE PIPE 语句创建持续导入管道：

```sql
CREATE PIPE ods.kafka_orders_pipe
COMMENT = '从 Kafka 持续导入订单数据到 ods.orders'
AS
INSERT INTO ods.orders
SELECT
    GET_JSON_OBJECT(kafka_value, '$.order_id')                    AS order_id,
    GET_JSON_OBJECT(kafka_value, '$.user_id')                     AS user_id,
    GET_JSON_OBJECT(kafka_value, '$.product_id')                  AS product_id,
    CAST(GET_JSON_OBJECT(kafka_value, '$.quantity') AS INT)       AS quantity,
    CAST(GET_JSON_OBJECT(kafka_value, '$.amount') AS DECIMAL(18,2)) AS amount,
    GET_JSON_OBJECT(kafka_value, '$.status')                      AS status,
    TO_TIMESTAMP(GET_JSON_OBJECT(kafka_value, '$.created_at'))    AS created_at,
    TO_TIMESTAMP(GET_JSON_OBJECT(kafka_value, '$.updated_at'))    AS updated_at
FROM KAFKA (
    'kafka.bootstrap.servers' = 'kafka.prod.com:9092',
    'kafka.topic'             = 'orders',
    'kafka.value.format'      = 'json',
    'kafka.group.id'          = 'clickzetta_orders_consumer',
    'kafka.auto.offset.reset' = 'earliest'
);
```

---

## 第三步：启动 PIPE

```sql
ALTER PIPE ods.kafka_orders_pipe RESUME;
```

---

## 第四步：查看 PIPE 状态

```sql
-- 查看 PIPE 运行状态
SHOW PIPES IN SCHEMA ods;

-- 查看指定 PIPE 详情
DESCRIBE PIPE ods.kafka_orders_pipe;
```

---

## 第五步：暂停或删除 PIPE（按需）

```sql
-- 暂停 PIPE
ALTER PIPE ods.kafka_orders_pipe SUSPEND;

-- 删除 PIPE
DROP PIPE ods.kafka_orders_pipe;
```

---

## 关键参数说明

| 参数 | 值 | 说明 |
|------|----|------|
| kafka.bootstrap.servers | kafka.prod.com:9092 | Kafka broker 地址 |
| kafka.topic | orders | 消费的 Topic 名称 |
| kafka.value.format | json | 消息格式为 JSON |
| kafka.group.id | clickzetta_orders_consumer | 消费者组 ID，可自定义 |
| kafka.auto.offset.reset | earliest | 从最早的消息开始消费；改为 latest 则从最新消息开始 |

---

## 注意事项

1. **字段映射**：上述 SELECT 中的字段需要根据实际 Kafka 消息的 JSON 结构进行调整。
2. **消费者组 ID**：kafka.group.id 建议使用有意义的名称，避免与其他消费者冲突。
3. **Offset 策略**：
   - earliest：从 Topic 最早的消息开始消费（适合首次导入历史数据）
   - latest：只消费 PIPE 启动后的新消息
4. **安全认证**：如果 Kafka 集群启用了 SASL/SSL 认证，需要额外添加认证相关参数，例如：
   ```sql
   'kafka.security.protocol' = 'SASL_SSL',
   'kafka.sasl.mechanism'    = 'PLAIN',
   'kafka.sasl.username'     = 'your_username',
   'kafka.sasl.password'     = 'your_password'
   ```
5. **幂等性**：如果需要防止重复数据，建议在目标表上设置主键约束，或在 INSERT 时使用 MERGE 语句。
6. **监控**：建议定期查看 PIPE 状态，确保数据持续正常导入。
