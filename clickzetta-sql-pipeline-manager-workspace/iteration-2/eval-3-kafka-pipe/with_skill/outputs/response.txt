根据你的需求，使用 **Pipe** 对象从 Kafka 持续导入订单数据到 `ods.orders` 表。

## 步骤 0：确认连接

确保已连接到 ClickZetta Lakehouse（参考 `clickzetta-lakehouse-connect` skill）。

---

## 步骤 1：（可选）创建前验证 Kafka 连接

在创建 Pipe 之前，建议先用 `READ_KAFKA` 函数验证连接是否正常、数据格式是否符合预期：

```sql
SELECT *
FROM TABLE(
  READ_KAFKA(
    KAFKA_BROKER     => 'kafka.prod.com:9092',
    KAFKA_TOPIC      => 'orders',
    KAFKA_GROUP_ID   => 'test_group',
    KAFKA_OFFSET     => 'earliest',
    KAFKA_DATA_FORMAT => 'json'
  )
)
LIMIT 10;
```

观察返回的 JSON 字段，确认字段名和类型后再创建 Pipe。

---

## 步骤 2：创建 Pipe

由于你的数据是 JSON 格式，使用 `$1:field_name::TYPE` 语法从 JSON 中提取字段。

以下 SQL 假设 `ods.orders` 表已存在，且包含常见订单字段（order_id、user_id、amount、status、created_at）。请根据实际表结构调整字段列表和类型。

```sql
-- 创建 Pipe，从 Kafka 持续导入订单数据到 ods.orders
CREATE OR REPLACE PIPE kafka_orders_pipe
  COMMENT '从 Kafka orders topic 持续导入订单数据到 ods.orders'
AS
INSERT INTO ods.orders (order_id, user_id, amount, status, created_at)
SELECT
  $1:order_id::STRING,
  $1:user_id::STRING,
  $1:amount::DECIMAL(10,2),
  $1:status::STRING,
  $1:created_at::TIMESTAMP
FROM TABLE(
  READ_KAFKA(
    KAFKA_BROKER      => 'kafka.prod.com:9092',
    KAFKA_TOPIC       => 'orders',
    KAFKA_GROUP_ID    => 'lakehouse_orders_ingest',
    KAFKA_OFFSET      => 'latest',
    KAFKA_DATA_FORMAT => 'json'
  )
);
```

**参数说明：**
- `KAFKA_BROKER`：你提供的 broker 地址 `kafka.prod.com:9092`
- `KAFKA_TOPIC`：你提供的 topic 名称 `orders`
- `KAFKA_GROUP_ID`：消费者组 ID，可自定义，建议使用有意义的名称
- `KAFKA_OFFSET`：`latest` 表示从最新消息开始消费（如需从头消费改为 `earliest`）
- `KAFKA_DATA_FORMAT`：`json`，与你的数据格式一致

> **注意**：Pipe 创建后默认自动启动，无需手动执行 RESUME。

## 步骤 3：验证 Pipe 状态

```sql
-- 查看所有 Pipe，确认 kafka_orders_pipe 已创建
SHOW PIPES;

-- 查看 Pipe 详情（状态、最近摄入信息等）
DESC PIPE kafka_orders_pipe;
```

## 常用运维操作

```sql
-- 暂停 Pipe（如需维护）
ALTER PIPE kafka_orders_pipe PAUSE;

-- 恢复 Pipe
ALTER PIPE kafka_orders_pipe RESUME;

-- 删除 Pipe
DROP PIPE IF EXISTS kafka_orders_pipe;
```

---

## 注意事项

1. **目标表需提前创建**：Pipe 不会自动建表，请确保 `ods.orders` 表已存在且字段与 SELECT 列表匹配。
2. **字段映射**：上面的字段列表（order_id、user_id、amount、status、created_at）是示例，请根据你的实际 JSON 消息结构和目标表结构调整。
3. **Consumer Group**：`KAFKA_GROUP_ID` 用于管理消费 offset，重建 Pipe 时注意 group_id 和 offset 设置，避免重复消费或跳过数据。
4. **认证**：如果 Kafka 需要 SASL 认证，在 `READ_KAFKA` 中补充 `KAFKA_SASL_USERNAME` 和 `KAFKA_SASL_PASSWORD` 参数。
