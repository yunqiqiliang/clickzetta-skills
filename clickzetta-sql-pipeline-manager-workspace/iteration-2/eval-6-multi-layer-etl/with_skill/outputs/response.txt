## 三层 ETL 动态表设计方案：ODS → DWD → DWS

### 设计思路

根据你的需求：
- ODS 原始层：已有数据库表（数据来源）
- DWD 清洗层：每分钟刷新 → `TARGET_LAG = '1 minutes'`
- DWS 聚合层：跟随 DWD 刷新 → `TARGET_LAG = DOWNSTREAM`

这是多层动态表级联的标准模式。DWS 设置 `DOWNSTREAM` 后，每次 DWD 刷新完成，DWS 会自动触发刷新，无需独立调度。

---

### 完整 SQL

以订单数据为例（根据你的实际字段替换）：

```sql
-- ============================================================
-- Step 0: 创建各层 Schema（如已存在可跳过）
-- ============================================================
CREATE SCHEMA IF NOT EXISTS ods;
CREATE SCHEMA IF NOT EXISTS dwd;
CREATE SCHEMA IF NOT EXISTS dws;

-- ============================================================
-- Step 1: ODS 原始层建表
-- 直接存储原始数据，不做任何转换
-- ============================================================
CREATE TABLE IF NOT EXISTS ods.orders (
  order_id    STRING,
  user_id     STRING,
  amount      DECIMAL(10, 2),
  status      STRING,
  region      STRING,
  created_at  TIMESTAMP
);

-- ============================================================
-- Step 2: DWD 清洗层动态表（每分钟刷新）
-- 过滤无效数据、标准化字段
-- ============================================================
CREATE OR REPLACE DYNAMIC TABLE dwd.orders_clean
  TARGET_LAG = '1 minutes'
  VCLUSTER = default_ap
  COMMENT = 'DWD 清洗层：过滤无效订单，标准化状态字段'
AS
SELECT
  order_id,
  user_id,
  amount,
  UPPER(status)        AS status,       -- 状态统一大写
  region,
  created_at,
  DATE(created_at)     AS dt            -- 分区字段
FROM ods.orders
WHERE amount > 0                        -- 过滤金额异常
  AND order_id IS NOT NULL;             -- 过滤空主键

-- ============================================================
-- Step 3: DWS 聚合层动态表（跟随 DWD 刷新）
-- 按小时 + 地区聚合，供 BI 查询
-- ============================================================
CREATE OR REPLACE DYNAMIC TABLE dws.order_stats_hourly
  TARGET_LAG = DOWNSTREAM
  VCLUSTER = default_ap
  COMMENT = 'DWS 聚合层：按小时+地区统计订单量和金额'
AS
SELECT
  DATE_TRUNC('hour', created_at)  AS stat_hour,
  region,
  status,
  COUNT(*)                        AS order_cnt,
  SUM(amount)                     AS total_amount,
  COUNT(DISTINCT user_id)         AS uv
FROM dwd.orders_clean
GROUP BY 1, 2, 3;
```

---

### 刷新链路说明

```
ODS（静态表，持续写入）
    ↓
DWD orders_clean   TARGET_LAG = '1 minutes'   ← 每分钟检查 ODS 是否有新数据，有则增量刷新
    ↓
DWS order_stats_hourly   TARGET_LAG = DOWNSTREAM   ← DWD 每次刷新完成后自动触发
```

- DWD 每分钟最多刷新一次，如果 ODS 没有新数据，跳过本次刷新
- DWS 的 `DOWNSTREAM` 意味着它不独立计时，完全由 DWD 驱动，端到端延迟约 1 分钟出头

### 如果 ODS 数据来自 Kafka

如果原始数据是从 Kafka 实时摄入，在 Step 1 之前加一个 Pipe：

```sql
-- Pipe 持续将 Kafka 数据写入 ODS
CREATE OR REPLACE PIPE ods.orders_kafka_pipe AS
INSERT INTO ods.orders (order_id, user_id, amount, status, region, created_at)
SELECT
  $1:order_id::STRING,
  $1:user_id::STRING,
  $1:amount::DECIMAL(10, 2),
  $1:status::STRING,
  $1:region::STRING,
  $1:created_at::TIMESTAMP
FROM TABLE(
  READ_KAFKA(
    KAFKA_BROKER     => 'kafka.example.com:9092',
    KAFKA_TOPIC      => 'orders',
    KAFKA_GROUP_ID   => 'lakehouse_ods_ingest',
    KAFKA_OFFSET     => 'latest',
    KAFKA_DATA_FORMAT => 'json'
  )
);
```

ODS 表结构不变，DWD 和 DWS 的动态表 SQL 完全复用。

### 如果 ODS 有 UPDATE/DELETE（CDC 场景）

ODS 如果会有记录更新（如订单状态变更），需要在 ODS 上加 Table Stream，让 DWD 消费变更：

```sql
-- 在 ODS 表上创建 Stream 捕获 INSERT/UPDATE/DELETE
CREATE TABLE STREAM ods.orders_stream
  ON TABLE ods.orders
  MODE = STANDARD;

-- DWD 改为消费 Stream，只取最新状态
CREATE OR REPLACE DYNAMIC TABLE dwd.orders_clean
  TARGET_LAG = '1 minutes'
  VCLUSTER = default_ap
AS
SELECT
  order_id,
  user_id,
  amount,
  UPPER(status)    AS status,
  region,
  created_at,
  DATE(created_at) AS dt
FROM ods.orders_stream
WHERE _change_type IN ('insert', 'update_postimage')
  AND amount > 0
  AND order_id IS NOT NULL;

-- DWS 不变，仍然 DOWNSTREAM
```

### 验证命令

```sql
-- 查看所有动态表状态
SHOW TABLES WHERE is_dynamic = true;

-- 查看 DWD 刷新历史（确认每分钟在刷新）
SHOW DYNAMIC TABLE REFRESH HISTORY dwd.orders_clean LIMIT 10;

-- 查看 DWS 刷新历史（确认跟随 DWD）
SHOW DYNAMIC TABLE REFRESH HISTORY dws.order_stats_hourly LIMIT 10;

-- 查看动态表结构和调度配置
DESC DYNAMIC TABLE dwd.orders_clean;
DESC DYNAMIC TABLE dws.order_stats_hourly;

-- 抽查数据
SELECT * FROM dwd.orders_clean LIMIT 20;
SELECT * FROM dws.order_stats_hourly ORDER BY stat_hour DESC LIMIT 20;
```

### 运维操作

```sql
-- 暂停整条链路（如集群维护）
ALTER DYNAMIC TABLE dwd.orders_clean SUSPEND;
ALTER DYNAMIC TABLE dws.order_stats_hourly SUSPEND;

-- 恢复
ALTER DYNAMIC TABLE dwd.orders_clean RESUME;
ALTER DYNAMIC TABLE dws.order_stats_hourly RESUME;

-- 调整 DWD 刷新频率（如改为 30 秒）
ALTER DYNAMIC TABLE dwd.orders_clean SET TARGET_LAG = '30 seconds';

-- 修改 SQL 逻辑（必须用 CREATE OR REPLACE，不能用 ALTER）
CREATE OR REPLACE DYNAMIC TABLE dwd.orders_clean
  TARGET_LAG = '1 minutes'
  VCLUSTER = default_ap
AS
-- 新的查询逻辑...
;
```

---

### 关键注意事项

1. **VCLUSTER 写法**：必须用 `VCLUSTER = default_ap`，不能写 `WAREHOUSE`（Snowflake 写法在 ClickZetta 不生效）
2. **DOWNSTREAM 的前提**：DWS 的 `TARGET_LAG = DOWNSTREAM` 要求它的查询来源必须是动态表（即 DWD），不能是普通静态表
3. **修改 SQL 逻辑**：只能用 `CREATE OR REPLACE DYNAMIC TABLE`，`ALTER DYNAMIC TABLE` 不支持修改 AS 子句
4. **刷新失败不影响查询**：动态表刷新失败时，查询返回上次成功版本的数据，不会报错
