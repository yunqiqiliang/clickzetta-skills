根据你的需求，使用 ClickZetta Pipe 从 Kafka 持续导入订单数据到 ods.orders 表。

由于你没有提供目标表的字段结构，以下 SQL 基于订单数据的常见字段生成。如果你的 JSON 消息包含不同字段，请相应调整 SELECT 中的字段映射。

---

## 步骤 0：确认连接

确保已连接到 ClickZetta Lakehouse 实例。

---

## 步骤 1：（可选）验证 Kafka 连接和数据格式

在创建 Pipe 之前，先用 READ_KAFKA 函数验证连接是否正常，并查看实际的 JSON 数据结构：

```sql
SELECT *
FROM TABLE(
  READ_KAFKA(
    KAFKA_BROKER     => 'kafka.prod.com:9092',
    KAFKA_TOPIC      => 'orders',
    KAFKA_GROUP_ID   => 'test_group',
    KAFKA_OFFSET     => 'earliest',
    KAFKA_DATA_FORMAT => 'json'
  )
)
LIMIT 10;
```

查看返回结果，确认 JSON 字段名称，然后再创建 Pipe。

---

## 步骤 2：确保目标表存在

如果 ods.orders 表尚未创建，先建表（根据实际字段调整）：

```sql
CREATE SCHEMA IF NOT EXISTS ods;

CREATE TABLE IF NOT EXISTS ods.orders (
  order_id   STRING,
  user_id    STRING,
  amount     DECIMAL(10, 2),
  status     STRING,
  created_at TIMESTAMP
);
```

---

## 步骤 3：创建 Pipe 持续导入 Kafka 数据

```sql
CREATE OR REPLACE PIPE kafka_orders_pipe
  COMMENT '从 Kafka 持续导入订单数据到 ods.orders'
AS
INSERT INTO ods.orders (order_id, user_id, amount, status, created_at)
SELECT
  $1:order_id::STRING,
  $1:user_id::STRING,
  $1:amount::DECIMAL(10, 2),
  $1:status::STRING,
  $1:created_at::TIMESTAMP
FROM TABLE(
  READ_KAFKA(
    KAFKA_BROKER      => 'kafka.prod.com:9092',
    KAFKA_TOPIC       => 'orders',
    KAFKA_GROUP_ID    => 'lakehouse_orders_consumer',
    KAFKA_OFFSET      => 'latest',
    KAFKA_DATA_FORMAT => 'json'
  )
);
```

说明：
- KAFKA_BROKER：你提供的 kafka.prod.com:9092
- KAFKA_TOPIC：orders
- KAFKA_GROUP_ID：消费者组名称，可自定义，建议使用有意义的名称
- KAFKA_OFFSET：latest 表示从最新消息开始消费；如需从头消费历史数据，改为 earliest
- KAFKA_DATA_FORMAT：json，与你的数据格式一致
- $1:field_name::TYPE：ClickZetta 的 JSON 字段提取语法，$1 表示整行 JSON 消息
- Pipe 创建后默认自动启动，无需手动 RESUME

## 步骤 4：验证 Pipe 状态

```sql
-- 查看所有 Pipe
SHOW PIPES;

-- 查看 Pipe 详情（包含状态、错误信息等）
DESC PIPE kafka_orders_pipe;

-- 验证数据是否已写入目标表

```

## 运维操作

```sql
-- 暂停 Pipe（如需维护）
ALTER PIPE kafka_orders_pipe PAUSE;

-- 恢复 Pipe
ALTER PIPE kafka_orders_pipe RESUME;

-- 删除 Pipe
DROP PIPE IF EXISTS kafka_orders_pipe;
```

---

## 注意事项

1. 字段映射：上面的 SQL 假设 JSON 消息包含 order_id、user_id、amount、status、created_at 字段。请根据步骤 1 中查看到的实际 JSON 结构调整字段名和类型。

2. 如果 Kafka 需要认证（SASL），在 READ_KAFKA 中添加：
   KAFKA_SASL_USERNAME => '你的用户名',
   KAFKA_SASL_PASSWORD => '你的密码'

3. 如果使用 Schema Registry（Avro 格式），添加：
   KAFKA_SCHEMA_REGISTRY_URL => 'http://schema-registry:8081'

4. 重建 Pipe 时注意：如果使用相同的 KAFKA_GROUP_ID，Kafka 会从上次提交的 offset 继续消费；如需重新消费，更换 group_id 并设置 KAFKA_OFFSET => 'earliest'。

5. Pipe 不支持 ALTER 修改 AS 子句，如需修改导入逻辑，使用 CREATE OR REPLACE PIPE 重建。
