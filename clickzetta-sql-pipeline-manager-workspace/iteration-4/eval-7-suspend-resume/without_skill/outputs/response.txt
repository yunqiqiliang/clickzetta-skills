在 ClickZetta Lakehouse 中，暂停和恢复动态表刷新的操作如下：

## 暂停所有动态表刷新（维护前）

首先查出当前 schema 下所有动态表：

```sql
SHOW TABLES WHERE is_dynamic = true;
```

然后对每张动态表执行暂停：

```sql
ALTER DYNAMIC TABLE <schema_name>.<table_name> SUSPEND;
```

如果有多张表，逐一执行，例如：

```sql
ALTER DYNAMIC TABLE dw.order_summary SUSPEND;
ALTER DYNAMIC TABLE dw.order_daily SUSPEND;
ALTER DYNAMIC TABLE dw.user_stats SUSPEND;
```

## 维护结束后恢复刷新

对每张动态表执行恢复：

```sql
ALTER DYNAMIC TABLE dw.order_summary RESUME;
ALTER DYNAMIC TABLE dw.order_daily RESUME;
ALTER DYNAMIC TABLE dw.user_stats RESUME;
```

## 注意事项

- SUSPEND 后动态表仍然可以正常查询，返回的是最后一次成功刷新的数据，不影响下游读取。
- RESUME 后动态表会按照原来的 TARGET_LAG 配置重新开始调度刷新。
- 如果动态表之间有依赖关系（下游表设置了 TARGET_LAG = DOWNSTREAM），暂停上游表即可阻止整条链路的刷新，但建议将所有表都显式 SUSPEND，避免歧义。
- 恢复时建议先 RESUME 上游表，再 RESUME 下游表，保证数据依赖顺序正确。
