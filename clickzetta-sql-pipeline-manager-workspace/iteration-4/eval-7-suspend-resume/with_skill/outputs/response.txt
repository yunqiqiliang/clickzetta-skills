## 暂停与恢复所有动态表的操作指南

维护期间暂停所有动态表刷新，维护结束后恢复，分两步操作：

---

### 第一步：维护前——暂停所有动态表

先查出当前所有动态表：

```sql
SHOW TABLES WHERE is_dynamic = true;
```

然后对每张动态表执行 SUSPEND：

```sql
-- 按实际表名逐一执行，例如：
ALTER DYNAMIC TABLE dwd.orders_clean SUSPEND;
ALTER DYNAMIC TABLE dws.order_hourly SUSPEND;
-- ... 其余动态表依此类推
```

SUSPEND 后，动态表停止自动刷新，已有数据保留不变，查询仍可正常返回（返回暂停前的最后一次刷新结果）。

---

### 第二步：维护结束后——恢复所有动态表

```sql
-- 按实际表名逐一执行，例如：
ALTER DYNAMIC TABLE dwd.orders_clean RESUME;
ALTER DYNAMIC TABLE dws.order_hourly RESUME;
-- ... 其余动态表依此类推
```

RESUME 后，动态表会立即触发一次增量刷新，追上维护期间积累的数据变更，之后恢复按 TARGET_LAG 设定的频率自动刷新。

---

### 验证状态

恢复后可通过以下命令确认刷新是否正常：

```sql
-- 查看所有动态表状态
SHOW TABLES WHERE is_dynamic = true;

-- 查看某张表的刷新历史，确认恢复后有新的成功记录
SHOW DYNAMIC TABLE REFRESH HISTORY dwd.orders_clean LIMIT 5;
```

---

### 注意事项

- 如果动态表之间有级联关系（下游表 TARGET_LAG = DOWNSTREAM），建议按从上游到下游的顺序 RESUME，确保数据依赖链路正确恢复。
- 如果同时有 Pipe 也需要暂停，使用：
  ```sql
  ALTER PIPE <pipe_name> PAUSE;   -- 维护前
  ALTER PIPE <pipe_name> RESUME;  -- 维护后
  ```
- SUSPEND/RESUME 操作不影响表结构和历史数据，仅控制自动刷新行为。
