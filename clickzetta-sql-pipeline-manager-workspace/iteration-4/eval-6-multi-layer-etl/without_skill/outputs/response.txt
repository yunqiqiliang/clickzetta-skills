三层 ETL 动态表设计方案（ClickZetta Lakehouse）

架构概览：
ODS（原始层）→ DWD（清洗层，每分钟刷新）→ DWS（聚合层，跟随 DWD 刷新）

---

第一层：ODS 原始层

ODS 层通常是外部表或普通表，直接承接数据源写入，不使用动态表。

-- 示例：ODS 原始表（普通表，由数据管道写入）
CREATE TABLE ods_orders_raw (
    order_id     STRING,
    user_id      STRING,
    product_id   STRING,
    amount       DOUBLE,
    status       STRING,
    created_at   TIMESTAMP,
    updated_at   TIMESTAMP,
    _ingest_time TIMESTAMP  -- 数据入湖时间
);

---

第二层：DWD 清洗层（动态表，每分钟刷新）

DWD 层对 ODS 数据做清洗、标准化、去重等处理，设置 TARGET_LAG = '1 minute'。

CREATE DYNAMIC TABLE dwd_orders_clean
    TARGET_LAG = '1 minute'
    WAREHOUSE = 'your_warehouse'
AS
SELECT
    order_id,
    user_id,
    product_id,
    COALESCE(amount, 0.0)                          AS amount,
    UPPER(TRIM(status))                            AS status,
    created_at,
    updated_at,
    DATE(created_at)                               AS order_date,
    YEAR(created_at)                               AS order_year,
    MONTH(created_at)                              AS order_month,
    _ingest_time
FROM ods_orders_raw
WHERE order_id IS NOT NULL
  AND user_id  IS NOT NULL
  AND amount   >= 0;

关键参数说明：
- TARGET_LAG = '1 minute'：DWD 最多落后上游 1 分钟，ClickZetta 会自动调度刷新。
- WAREHOUSE：指定执行刷新任务的计算资源。

---

第三层：DWS 聚合层（动态表，跟随 DWD 刷新）

DWS 层基于 DWD 动态表做聚合，TARGET_LAG 设置为 'DOWNSTREAM' 或与 DWD 相同/更宽松的值。

方案 A：TARGET_LAG = 'DOWNSTREAM'（推荐）
-- 表示 DWS 跟随其上游（DWD）的刷新节奏，DWD 刷新后 DWS 立即触发刷新。
-- 这是最紧密的联动方式，适合对时效性要求高的场景。

CREATE DYNAMIC TABLE dws_orders_daily_summary
    TARGET_LAG = 'DOWNSTREAM'
    WAREHOUSE = 'your_warehouse'
AS
SELECT
    order_date,
    order_year,
    order_month,
    status,
    COUNT(DISTINCT order_id)  AS order_count,
    COUNT(DISTINCT user_id)   AS user_count,
    SUM(amount)               AS total_amount,
    AVG(amount)               AS avg_amount,
    MAX(updated_at)           AS last_updated
FROM dwd_orders_clean
GROUP BY
    order_date,
    order_year,
    order_month,
    status;

方案 B：TARGET_LAG = '1 minute'（与 DWD 相同）
-- DWS 独立按 1 分钟调度，不强依赖 DWD 的刷新完成时间。
-- 适合 DWS 计算量大、希望错开刷新窗口的场景。

CREATE DYNAMIC TABLE dws_orders_daily_summary
    TARGET_LAG = '1 minute'
    WAREHOUSE = 'your_warehouse'
AS
SELECT ...  -- 同上

方案 A vs 方案 B 选择建议：
- 如果希望 DWS 数据与 DWD 严格一致（DWD 刷完才刷 DWS），用 DOWNSTREAM。
- 如果 DWS 聚合计算较重，希望有独立的调度控制，用固定 TARGET_LAG。

---

完整三层示例（串联）

-- Step 1: ODS（普通表，外部写入）
CREATE TABLE ods_events (
    event_id   STRING,
    user_id    STRING,
    event_type STRING,
    event_time TIMESTAMP,
    properties STRING  -- JSON 字符串
);

-- Step 2: DWD（动态表，每分钟清洗）
CREATE DYNAMIC TABLE dwd_events_clean
    TARGET_LAG = '1 minute'
    WAREHOUSE = 'etl_warehouse'
AS
SELECT
    event_id,
    user_id,
    LOWER(event_type)                        AS event_type,
    event_time,
    DATE(event_time)                         AS event_date,
    HOUR(event_time)                         AS event_hour,
    JSON_EXTRACT_STRING(properties, '$.page') AS page
FROM ods_events
WHERE event_id IS NOT NULL
  AND event_time >= DATEADD(day, -7, CURRENT_TIMESTAMP());  -- 只处理近 7 天

-- Step 3: DWS（动态表，跟随 DWD）
CREATE DYNAMIC TABLE dws_events_hourly
    TARGET_LAG = 'DOWNSTREAM'
    WAREHOUSE = 'etl_warehouse'
AS
SELECT
    event_date,
    event_hour,
    event_type,
    page,
    COUNT(DISTINCT user_id)  AS uv,
    COUNT(event_id)          AS pv
FROM dwd_events_clean
GROUP BY
    event_date,
    event_hour,
    event_type,
    page;

---

注意事项

1. ODS 层不建议用动态表
   ODS 是数据入口，通常由 Kafka、Flink、数据集成工具直接写入，用普通表或外部表即可。

2. TARGET_LAG 的含义
   TARGET_LAG 是"目标延迟上限"，不是固定刷新间隔。ClickZetta 会根据上游变化情况智能调度，
   实际刷新频率可能比设定值更频繁（有数据变化时）或更低（无变化时）。

3. DOWNSTREAM 的传播规则
   如果 DWS 设置了 DOWNSTREAM，而 DWD 设置了 1 minute，则整条链路的端到端延迟约为 1 分钟。
   如果有多层 DOWNSTREAM 嵌套，延迟会叠加，需注意。

4. 计算资源规划
   DWD 和 DWS 可以共用一个 warehouse，也可以分开，取决于计算量和资源隔离需求。
   高频刷新（1 分钟）对 warehouse 有持续负载，建议使用自动扩缩容配置。

5. 增量 vs 全量刷新
   ClickZetta 动态表支持增量刷新（INCREMENTAL）和全量刷新（FULL）两种模式。
   对于聚合类 DWS 表，通常是全量重算；对于明细类 DWD 表，可以配置增量模式以降低计算成本。
   具体模式由 ClickZetta 根据 SQL 语义自动判断，或可通过 REFRESH_MODE 参数手动指定。

---

总结

层级        | 表类型      | TARGET_LAG      | 说明
------------|-------------|-----------------|---------------------------
ODS         | 普通表      | 不适用          | 数据源直接写入
DWD         | 动态表      | 1 minute        | 清洗、标准化，每分钟刷新
DWS         | 动态表      | DOWNSTREAM      | 聚合，跟随 DWD 自动刷新
