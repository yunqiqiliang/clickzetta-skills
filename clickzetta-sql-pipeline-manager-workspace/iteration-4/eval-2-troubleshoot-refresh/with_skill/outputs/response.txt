动态表 dws.order_hourly 刷新失败排查指南

## 第一步：查看刷新历史，获取错误详情

这是最直接的排查入口，能看到失败原因、时间、触发方式：

```sql
SHOW DYNAMIC TABLE REFRESH HISTORY dws.order_hourly LIMIT 10;
```

重点关注输出中的 state（状态）和 error_message（错误信息）字段。

## 第二步：查看动态表当前配置

```sql
DESC DYNAMIC TABLE dws.order_hourly;
-- 或查看完整建表语句
SHOW CREATE TABLE dws.order_hourly;
```

确认以下几点：
- VCLUSTER 名称是否正确（ClickZetta 用 VCLUSTER，不是 WAREHOUSE）
- TARGET_LAG 设置是否合理
- 动态表当前状态是否为 SUSPENDED（已暂停）

## 第三步：根据错误类型逐一排查

### 常见错误及解决方案

**1. VCluster not available（计算集群不可用）**
- 原因：VCLUSTER 名称写错，或集群未启动
- 解决：确认集群名称，检查集群状态是否正常

**2. 源表结构变更导致 SQL 报错**
- 原因：dws.order_hourly 依赖的上游表（如 dwd.orders_clean）字段被修改或删除
- 解决：检查上游表结构，用 `CREATE OR REPLACE DYNAMIC TABLE` 重建动态表

```sql
-- 检查上游表结构
DESC TABLE dwd.orders_clean;
```

**3. 上游动态表本身也在失败**
- 如果 dws.order_hourly 设置了 TARGET_LAG = DOWNSTREAM，它依赖上游动态表触发
- 先排查上游动态表的刷新状态

```sql
-- 查看上游动态表刷新历史
SHOW DYNAMIC TABLE REFRESH HISTORY dwd.orders_clean LIMIT 10;
```

**4. 动态表被暂停**
- 检查是否被手动 SUSPEND 过

```sql
SHOW TABLES IN dws WHERE is_dynamic = true;
```

如果状态是 SUSPENDED，执行恢复：

```sql
ALTER DYNAMIC TABLE dws.order_hourly RESUME;
```

**5. SQL 查询逻辑本身有问题**
- 检查 AS 子句中的查询是否能独立执行

```sql
-- 单独运行动态表的查询逻辑，看是否报错
SELECT
  DATE_TRUNC('hour', created_at) AS hour,
  status,
  COUNT(*) AS order_cnt,
  SUM(amount) AS total_amount
FROM dwd.orders_clean
GROUP BY 1, 2;
```

## 第四步：修复后验证

修复问题后，确认刷新恢复正常：

```sql
-- 查看最新刷新历史，确认 state = SUCCESS
SHOW DYNAMIC TABLE REFRESH HISTORY dws.order_hourly LIMIT 5;
```

## 注意事项

- 刷新失败不影响表的可查询性，查询会返回上次成功刷新的数据版本
- 如果需要修改动态表的 SQL 逻辑，必须用 CREATE OR REPLACE，不能用 ALTER 修改 AS 子句：

```sql
-- 正确做法：修改 SQL 逻辑
CREATE OR REPLACE DYNAMIC TABLE dws.order_hourly
  TARGET_LAG = DOWNSTREAM
  VCLUSTER = default_ap
AS
SELECT
  DATE_TRUNC('hour', created_at) AS hour,
  status,
  COUNT(*) AS order_cnt,
  SUM(amount) AS total_amount
FROM dwd.orders_clean
GROUP BY 1, 2;
```

## 排查流程总结

1. SHOW DYNAMIC TABLE REFRESH HISTORY dws.order_hourly LIMIT 10; → 看错误信息
2. 根据错误类型定位：集群问题 / 源表结构变更 / 上游失败 / 被暂停 / SQL 报错
3. 修复对应问题
4. 如需改 SQL 逻辑，用 CREATE OR REPLACE DYNAMIC TABLE 重建
5. 验证刷新历史确认恢复正常
