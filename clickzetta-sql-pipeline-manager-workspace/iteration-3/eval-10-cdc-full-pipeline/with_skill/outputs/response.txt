根据你的需求分析：

- 数据来源：已有表 `ods.orders`，有 INSERT 和 UPDATE（需要 CDC）
- 字段：order_id, customer_id, amount, status, updated_at
- 层次：ODS（已有）→ DWD（最新状态去重）→ DWS（按天按 status 聚合）
- 延迟要求：近实时，1 分钟以内

**方案选择：**
- 因为源表有 UPDATE，使用 `Table Stream（MODE = STANDARD）` 捕获变更
- DWD 层用 Dynamic Table 消费 Stream，过滤 `update_postimage` 保留最新状态，`TARGET_LAG = '1 minutes'`
- DWS 层用 Dynamic Table 做聚合，`TARGET_LAG = DOWNSTREAM` 跟随 DWD 自动触发

---

## 完整 Pipeline SQL

### Step 1：创建 Schema

```sql
CREATE SCHEMA IF NOT EXISTS dwd;
CREATE SCHEMA IF NOT EXISTS dws;
```

### Step 2：在 ODS 表上创建 Table Stream 捕获 CDC 变更

```sql
-- 捕获 ods.orders 的所有变更（INSERT / UPDATE / DELETE）
CREATE TABLE STREAM ods.orders_stream
  ON TABLE ods.orders
  MODE = STANDARD
  COMMENT 'CDC 流，捕获 orders 表的 INSERT 和 UPDATE 变更';
```

### Step 3：DWD 层 —— 保留最新状态的订单

```sql
-- DWD 动态表：消费 Stream，只保留 insert 和 update_postimage（最新状态）
-- TARGET_LAG = '1 minutes' 保证近实时，延迟 ≤ 1 分钟
CREATE OR REPLACE DYNAMIC TABLE dwd.orders_latest
  TARGET_LAG = '1 minutes'
  VCLUSTER = default_ap
  COMMENT 'DWD 层：保留每条订单的最新状态（去除更新前镜像和删除记录）'
AS
SELECT
  order_id,
  customer_id,
  amount,
  status,
  updated_at,
  DATE(updated_at) AS dt
FROM ods.orders_stream
WHERE _change_type IN ('insert', 'update_postimage');
```

### Step 4：DWS 层 —— 按天统计每个 status 的订单数和金额

```sql
-- DWS 动态表：按天、按 status 聚合
-- TARGET_LAG = DOWNSTREAM 跟随 DWD 刷新，无需独立调度
CREATE OR REPLACE DYNAMIC TABLE dws.orders_daily_stats
  TARGET_LAG = DOWNSTREAM
  VCLUSTER = default_ap
  COMMENT 'DWS 层：按天、按订单状态统计订单数和总金额'
AS
SELECT
  dt,
  status,
  COUNT(*)        AS order_cnt,
  SUM(amount)     AS total_amount
FROM dwd.orders_latest
GROUP BY dt, status;
```

## 验证命令

```sql
-- 确认 Stream 已创建
SHOW TABLE STREAMS IN ods;

-- 确认动态表已创建
SHOW TABLES IN dwd WHERE is_dynamic = true;
SHOW TABLES IN dws WHERE is_dynamic = true;

-- 查看 DWD 刷新历史（确认是否正常增量刷新）
SHOW DYNAMIC TABLE REFRESH HISTORY dwd.orders_latest LIMIT 5;

-- 查看 DWS 刷新历史
SHOW DYNAMIC TABLE REFRESH HISTORY dws.orders_daily_stats LIMIT 5;

-- 查询验证数据
SELECT * FROM dwd.orders_latest LIMIT 20;
SELECT * FROM dws.orders_daily_stats ORDER BY dt DESC, status LIMIT 50;
```

## 运维操作

```sql
-- 暂停（如集群维护）
ALTER DYNAMIC TABLE dwd.orders_latest SUSPEND;
ALTER DYNAMIC TABLE dws.orders_daily_stats SUSPEND;

-- 恢复
ALTER DYNAMIC TABLE dwd.orders_latest RESUME;
ALTER DYNAMIC TABLE dws.orders_daily_stats RESUME;

-- 调整 DWD 刷新频率（如需更低延迟，改为 30 秒）
ALTER DYNAMIC TABLE dwd.orders_latest SET TARGET_LAG = '30 seconds';
```

## 架构说明

```
ods.orders（源表，持续 INSERT/UPDATE）
    │
    ▼
ods.orders_stream（Table Stream，MODE=STANDARD，捕获所有变更）
    │
    ▼
dwd.orders_latest（Dynamic Table，TARGET_LAG='1 minutes'）
    过滤 _change_type IN ('insert','update_postimage')
    保留最新状态，附加 dt 分区字段
    │
    ▼
dws.orders_daily_stats（Dynamic Table，TARGET_LAG=DOWNSTREAM）
    按 dt + status 聚合 order_cnt 和 total_amount
```

**延迟分析：**
- Stream 捕获变更：近实时（毫秒级）
- DWD 刷新：最大延迟 1 分钟（TARGET_LAG = '1 minutes'）
- DWS 刷新：DWD 刷新完成后立即触发（DOWNSTREAM），额外延迟极小
- 端到端总延迟：≤ 1 分钟，满足需求

**关键注意事项：**
1. Stream 数据一次性消费，Dynamic Table 会自动管理消费状态，无需手动处理
2. `update_postimage` 是 UPDATE 后的新值，`update_preimage` 是旧值，DWD 只需要 postimage
3. 如果业务有 DELETE 操作需要在 DWD 中体现，可在 WHERE 条件中加入 `'delete'` 并在下游做相应处理
4. DWS 的 `total_amount` 基于 DWD 当前快照聚合，已自动反映最新状态（无需手动去重）
