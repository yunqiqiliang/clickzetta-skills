# ClickZetta 数仓管道：CDC 全链路方案（ODS → DWD → DWS）

## 架构概述

```
业务数据库 (INSERT/UPDATE)
    ↓ CDC 同步
ods.orders（原始增量表）
    ↓ 近实时合并（~1分钟）
dwd.orders_latest（最新状态订单表）
    ↓ 近实时聚合（~1分钟）
dws.orders_daily_status（按天按状态统计）
```

## 第二步：DWD 层 —— 保留最新状态的订单

### 2.1 创建 DWD 目标表

```sql
CREATE TABLE IF NOT EXISTS dwd.orders_latest (
    order_id    BIGINT        NOT NULL,
    customer_id BIGINT,
    amount      DECIMAL(18, 2),
    status      VARCHAR(50),
    updated_at  TIMESTAMP,
    dwd_updated_at TIMESTAMP  -- 记录 DWD 层最后更新时间
)
USING delta
TBLPROPERTIES (
    'delta.enableChangeDataFeed' = 'true'
);
```

### 2.2 DWD 层近实时刷新任务

使用 MERGE INTO 实现 upsert，保留每个 order_id 的最新记录：

```sql
-- DWD 刷新任务（每分钟调度执行）
MERGE INTO dwd.orders_latest AS target
USING (
    -- 从 ODS 取每个 order_id 的最新一条记录
    SELECT
        order_id,
        customer_id,
        amount,
        status,
        updated_at
    FROM (
        SELECT
            order_id,
            customer_id,
            amount,
            status,
            updated_at,
            ROW_NUMBER() OVER (
                PARTITION BY order_id
                ORDER BY updated_at DESC
            ) AS rn
        FROM ods.orders
        WHERE updated_at >= CURRENT_TIMESTAMP - INTERVAL 5 MINUTES
          -- 适当扩大窗口以防调度延迟导致数据遗漏
    ) t
    WHERE rn = 1
) AS source
ON target.order_id = source.order_id
WHEN MATCHED AND source.updated_at > target.updated_at THEN
    UPDATE SET
        target.customer_id    = source.customer_id,
        target.amount         = source.amount,
        target.status         = source.status,
        target.updated_at     = source.updated_at,
        target.dwd_updated_at = CURRENT_TIMESTAMP
WHEN NOT MATCHED THEN
    INSERT (order_id, customer_id, amount, status, updated_at, dwd_updated_at)
    VALUES (source.order_id, source.customer_id, source.amount,
            source.status, source.updated_at, CURRENT_TIMESTAMP);
```

> 说明：
> - `WHERE updated_at >= CURRENT_TIMESTAMP - INTERVAL 5 MINUTES` 使用滑动窗口，保证调度抖动时不丢数据。
> - `WHEN MATCHED AND source.updated_at > target.updated_at` 防止乱序数据覆盖更新的记录。

## 第三步：DWS 层 —— 按天统计每个 status 的订单数和金额

### 3.1 创建 DWS 目标表

```sql
CREATE TABLE IF NOT EXISTS dws.orders_daily_status (
    stat_date   DATE          NOT NULL,
    status      VARCHAR(50)   NOT NULL,
    order_cnt   BIGINT,
    total_amount DECIMAL(18, 2),
    dws_updated_at TIMESTAMP
)
USING delta;
```

### 3.2 DWS 层近实时刷新任务

基于 DWD 层数据，按天、按 status 聚合，使用 MERGE INTO 更新当天及近期数据：

```sql
-- DWS 刷新任务（每分钟调度执行）
MERGE INTO dws.orders_daily_status AS target
USING (
    -- 从 DWD 层聚合：只重算最近有变化的日期
    SELECT
        CAST(updated_at AS DATE)  AS stat_date,
        status,
        COUNT(*)                  AS order_cnt,
        SUM(amount)               AS total_amount
    FROM dwd.orders_latest
    WHERE CAST(updated_at AS DATE) >= CURRENT_DATE - INTERVAL 2 DAYS
      -- 覆盖今天和昨天，防止跨天边界问题
    GROUP BY
        CAST(updated_at AS DATE),
        status
) AS source
ON  target.stat_date = source.stat_date
AND target.status    = source.status
WHEN MATCHED THEN
    UPDATE SET
        target.order_cnt      = source.order_cnt,
        target.total_amount   = source.total_amount,
        target.dws_updated_at = CURRENT_TIMESTAMP
WHEN NOT MATCHED THEN
    INSERT (stat_date, status, order_cnt, total_amount, dws_updated_at)
    VALUES (source.stat_date, source.status, source.order_cnt,
            source.total_amount, CURRENT_TIMESTAMP);
```

> 说明：
> - DWS 层基于 DWD 层全量聚合当天数据，保证幂等性——重复执行结果一致。
> - `INTERVAL 2 DAYS` 覆盖跨天边界，避免昨天最后一批数据被遗漏。

## 第四步：调度配置

在 ClickZetta 中使用 Task（定时任务）每分钟触发：

```sql
-- 创建 DWD 刷新 Task
CREATE TASK IF NOT EXISTS task_dwd_orders_refresh
    WAREHOUSE = 'your_warehouse'
    SCHEDULE  = '1 MINUTE'
AS
-- （粘贴上方 DWD MERGE INTO 语句）
MERGE INTO dwd.orders_latest AS target
USING (
    SELECT order_id, customer_id, amount, status, updated_at
    FROM (
        SELECT *, ROW_NUMBER() OVER (PARTITION BY order_id ORDER BY updated_at DESC) AS rn
        FROM ods.orders
        WHERE updated_at >= CURRENT_TIMESTAMP - INTERVAL 5 MINUTES
    ) t WHERE rn = 1
) AS source
ON target.order_id = source.order_id
WHEN MATCHED AND source.updated_at > target.updated_at THEN
    UPDATE SET
        customer_id = source.customer_id, amount = source.amount,
        status = source.status, updated_at = source.updated_at,
        dwd_updated_at = CURRENT_TIMESTAMP
WHEN NOT MATCHED THEN
    INSERT (order_id, customer_id, amount, status, updated_at, dwd_updated_at)
    VALUES (source.order_id, source.customer_id, source.amount,
            source.status, source.updated_at, CURRENT_TIMESTAMP);

-- 创建 DWS 刷新 Task（依赖 DWD Task 完成后触发）
CREATE TASK IF NOT EXISTS task_dws_orders_refresh
    WAREHOUSE  = 'your_warehouse'
    AFTER      task_dwd_orders_refresh
AS
MERGE INTO dws.orders_daily_status AS target
USING (
    SELECT CAST(updated_at AS DATE) AS stat_date, status,
           COUNT(*) AS order_cnt, SUM(amount) AS total_amount
    FROM dwd.orders_latest
    WHERE CAST(updated_at AS DATE) >= CURRENT_DATE - INTERVAL 2 DAYS
    GROUP BY CAST(updated_at AS DATE), status
) AS source
ON target.stat_date = source.stat_date AND target.status = source.status
WHEN MATCHED THEN
    UPDATE SET order_cnt = source.order_cnt, total_amount = source.total_amount,
               dws_updated_at = CURRENT_TIMESTAMP
WHEN NOT MATCHED THEN
    INSERT (stat_date, status, order_cnt, total_amount, dws_updated_at)
    VALUES (source.stat_date, source.status, source.order_cnt,
            source.total_amount, CURRENT_TIMESTAMP);

-- 启动任务
ALTER TASK task_dwd_orders_refresh RESUME;
ALTER TASK task_dws_orders_refresh RESUME;
```

## 延迟分析

| 阶段 | 延迟来源 | 预估延迟 |
|------|----------|----------|
| 业务DB → ODS | CDC 工具（Flink CDC / Debezium） | 秒级 |
| ODS → DWD | Task 调度间隔 + MERGE 执行时间 | < 60s |
| DWD → DWS | AFTER 依赖触发 + MERGE 执行时间 | < 30s |
| 端到端总延迟 | | < 90s（通常 < 60s）|

> 若要严格保证端到端 < 1 分钟，可将 Task 调度间隔缩短至 30 秒，并优化 ODS 表的 updated_at 字段索引。

---

## 关键设计要点

1. **幂等性**：MERGE INTO 保证重复执行不产生重复数据，任务失败重跑安全。
2. **防乱序**：DWD MERGE 中 `source.updated_at > target.updated_at` 条件防止旧数据覆盖新数据。
3. **滑动窗口**：ODS 读取使用 5 分钟滑动窗口，容忍调度抖动和轻微延迟。
4. **跨天覆盖**：DWS 聚合覆盖最近 2 天，避免跨天边界数据遗漏。
5. **任务依赖**：DWS Task 使用 `AFTER` 依赖 DWD Task，保证数据一致性。
