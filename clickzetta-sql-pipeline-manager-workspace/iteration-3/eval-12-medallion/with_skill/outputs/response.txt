以下是在 ClickZetta 上实现 Medallion Architecture（Bronze / Silver / Gold 三层）的完整 SQL 方案。

数据来源：S3 Volume（s3_raw_vol），路径 /events/，JSON 格式。
字段：event_id、user_id、event_name、properties（JSON 嵌套，含 page_url 和 referrer）、created_at。

---

## 架构说明

- Bronze 层：Pipe 持续从 S3 导入原始 JSON 数据，落地到 bronze.events_raw 表
- Silver 层：Dynamic Table，每 5 分钟刷新，过滤 event_name 为空的记录，展开 properties 中的 page_url 和 referrer
- Gold 层：Dynamic Table，TARGET_LAG = DOWNSTREAM，跟随 Silver 自动触发，按天统计每个 event_name 的次数和独立用户数

---

## 完整 SQL

### Step 1：创建 Schema

```sql
CREATE SCHEMA IF NOT EXISTS bronze;
CREATE SCHEMA IF NOT EXISTS silver;
CREATE SCHEMA IF NOT EXISTS gold;
```

---

### Step 2：Bronze 层 — 创建原始数据落地表

```sql
CREATE TABLE IF NOT EXISTS bronze.events_raw (
  event_id    STRING,
  user_id     STRING,
  event_name  STRING,
  properties  VARIANT,
  created_at  TIMESTAMP,
  _loaded_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP()
);
```

---

### Step 3：Bronze 层 — 创建 Pipe 持续从 S3 导入

```sql
CREATE OR REPLACE PIPE bronze.s3_events_pipe
  AUTO_INGEST = TRUE
  COMMENT 'S3 events JSON 持续导入到 Bronze 层'
AS
COPY INTO bronze.events_raw (event_id, user_id, event_name, properties, created_at)
FROM (
  SELECT
    $1:event_id::STRING,
    $1:user_id::STRING,
    $1:event_name::STRING,
    $1:properties::VARIANT,
    $1:created_at::TIMESTAMP
  FROM '@s3_raw_vol/events/'
)
FILE_FORMAT = ( TYPE = 'json' );
```

---

### Step 4：Silver 层 — 动态表（每 5 分钟刷新，清洗 + 展开）

```sql
CREATE OR REPLACE DYNAMIC TABLE silver.events_clean
  TARGET_LAG = '5 minutes'
  VCLUSTER = default_ap
  COMMENT 'Silver 层：过滤空 event_name，展开 properties 中的 page_url 和 referrer'
AS
SELECT
  event_id,
  user_id,
  event_name,
  properties:page_url::STRING   AS page_url,
  properties:referrer::STRING   AS referrer,
  created_at,
  DATE(created_at)              AS dt
FROM bronze.events_raw
WHERE event_name IS NOT NULL
  AND event_name != '';
```

---

### Step 5：Gold 层 — 动态表（跟随 Silver，按天聚合）

```sql
CREATE OR REPLACE DYNAMIC TABLE gold.events_daily_stats
  TARGET_LAG = DOWNSTREAM
  VCLUSTER = default_ap
  COMMENT 'Gold 层：按天统计每个 event_name 的次数和独立用户数'
AS
SELECT
  dt,
  event_name,
  COUNT(*)              AS event_count,
  COUNT(DISTINCT user_id) AS unique_users
FROM silver.events_clean
GROUP BY dt, event_name;
```

---

### Step 6：验证命令

```sql
-- 验证 Pipe 状态
SHOW PIPES;
DESC PIPE bronze.s3_events_pipe;

-- 验证 Bronze 表数据已导入
SELECT COUNT(*) FROM bronze.events_raw;

-- 验证动态表已创建
SHOW TABLES IN silver WHERE is_dynamic = true;
SHOW TABLES IN gold   WHERE is_dynamic = true;

-- 查看 Silver 层刷新历史
SHOW DYNAMIC TABLE REFRESH HISTORY silver.events_clean LIMIT 5;

-- 查看 Gold 层刷新历史
SHOW DYNAMIC TABLE REFRESH HISTORY gold.events_daily_stats LIMIT 5;

-- 抽查 Silver 层数据
SELECT * FROM silver.events_clean LIMIT 10;

-- 抽查 Gold 层聚合结果
SELECT * FROM gold.events_daily_stats ORDER BY dt DESC, event_count DESC LIMIT 20;
```

---

### Step 7：运维操作（按需使用）

```sql
-- 暂停 Silver 动态表（如需维护）
ALTER DYNAMIC TABLE silver.events_clean SUSPEND;

-- 恢复 Silver 动态表
ALTER DYNAMIC TABLE silver.events_clean RESUME;

-- 暂停 Gold 动态表
ALTER DYNAMIC TABLE gold.events_daily_stats SUSPEND;

-- 恢复 Gold 动态表
ALTER DYNAMIC TABLE gold.events_daily_stats RESUME;

-- 暂停 Pipe（停止 S3 导入）
ALTER PIPE bronze.s3_events_pipe PAUSE;

-- 恢复 Pipe
ALTER PIPE bronze.s3_events_pipe RESUME;

-- 修改 Silver 刷新频率（如需调整）
ALTER DYNAMIC TABLE silver.events_clean SET TARGET_LAG = '10 minutes';
```

## 架构总结

```
S3 (s3_raw_vol/events/*.json)
        │
        ▼  [Pipe: bronze.s3_events_pipe, AUTO_INGEST]
bronze.events_raw          ← 原始 JSON 落地，保留所有字段
        │
        ▼  [Dynamic Table, TARGET_LAG = '5 minutes']
silver.events_clean        ← 过滤 event_name 为空，展开 page_url / referrer
        │
        ▼  [Dynamic Table, TARGET_LAG = DOWNSTREAM]
gold.events_daily_stats    ← 按天 × event_name 统计 event_count / unique_users
```

**关键设计决策：**
- Pipe 使用 `AUTO_INGEST = TRUE`，S3 有新文件时自动触发导入，无需外部调度
- Silver 设置 `TARGET_LAG = '5 minutes'`，每 5 分钟增量刷新一次
- Gold 设置 `TARGET_LAG = DOWNSTREAM`，Silver 刷新完成后自动触发 Gold 刷新，保持数据一致性
- `properties` 字段存为 VARIANT 类型，用 `properties:page_url::STRING` 语法提取嵌套字段
- VCLUSTER 使用默认集群 `default_ap`，可按需替换为实际集群名称
