根据你提供的信息，以下是完整的三层数据管道 SQL，覆盖 ODS → DWD → DWS，数据来源为 Kafka，DWD 每 2 分钟刷新，DWS 跟随 DWD。

---

## 第一步：创建 Schema

```sql
CREATE SCHEMA IF NOT EXISTS ods;
CREATE SCHEMA IF NOT EXISTS dwd;
CREATE SCHEMA IF NOT EXISTS dws;
```

---

## 第二步：ODS 原始层 — 建表 + Pipe 接入 Kafka

先创建 ODS 原始表，用于落地 Kafka 消息：

```sql
-- ODS 原始表
CREATE TABLE IF NOT EXISTS ods.user_events (
  user_id    STRING,
  event_type STRING,
  page_id    STRING,
  ts         TIMESTAMP
);
```

创建 Pipe，持续从 Kafka 摄入数据到 ODS 表：

```sql
CREATE OR REPLACE PIPE ods.kafka_user_events_pipe AS
INSERT INTO ods.user_events (user_id, event_type, page_id, ts)
SELECT
  $1:user_id::STRING,
  $1:event_type::STRING,
  $1:page_id::STRING,
  $1:ts::TIMESTAMP
FROM TABLE(
  READ_KAFKA(
    KAFKA_BROKER      => 'kafka.prod.com:9092',
    KAFKA_TOPIC       => 'user_events',
    KAFKA_GROUP_ID    => 'lakehouse_user_events_ingest',
    KAFKA_OFFSET      => 'latest',
    KAFKA_DATA_FORMAT => 'json'
  )
);
```

---

## 第三步：DWD 清洗层 — 过滤无效 event_type

DWD 动态表从 ODS 表读取数据，过滤掉 event_type 为空或非法的记录，每 2 分钟增量刷新：

```sql
-- 有效 event_type 白名单可根据业务调整
CREATE OR REPLACE DYNAMIC TABLE dwd.user_events_clean
  TARGET_LAG = '2 minutes'
  VCLUSTER = default_ap
AS
SELECT
  user_id,
  event_type,
  page_id,
  ts,
  DATE(ts) AS dt
FROM ods.user_events
WHERE event_type IS NOT NULL
  AND event_type != ''
  AND event_type IN ('click', 'view', 'scroll', 'submit', 'search');
  -- 根据实际业务调整白名单，或改为黑名单过滤：AND event_type NOT IN ('unknown', 'invalid')
```

---

## 第四步：DWS 聚合层 — 按天统计每个 page 的 UV 和 PV

DWS 动态表从 DWD 层读取，按天、按 page_id 聚合 UV（去重用户数）和 PV（总访问次数），TARGET_LAG 设为 DOWNSTREAM 跟随 DWD 自动触发：

```sql
CREATE OR REPLACE DYNAMIC TABLE dws.page_daily_stats
  TARGET_LAG = DOWNSTREAM
  VCLUSTER = default_ap
AS
SELECT
  dt,
  page_id,
  COUNT(DISTINCT user_id) AS uv,
  COUNT(*)                AS pv
FROM dwd.user_events_clean
GROUP BY dt, page_id;
```

## 第五步：验证命令

```sql
-- 确认 Pipe 已创建并运行
SHOW PIPES;

-- 确认动态表已创建
SHOW TABLES WHERE is_dynamic = true;

-- 查看 DWD 刷新历史
SHOW DYNAMIC TABLE REFRESH HISTORY dwd.user_events_clean LIMIT 5;

-- 查看 DWS 刷新历史
SHOW DYNAMIC TABLE REFRESH HISTORY dws.page_daily_stats LIMIT 5;

-- 抽查 ODS 数据是否正常落地
SELECT * FROM ods.user_events LIMIT 10;

-- 抽查 DWD 清洗结果
SELECT * FROM dwd.user_events_clean LIMIT 10;

-- 抽查 DWS 聚合结果
SELECT * FROM dws.page_daily_stats ORDER BY dt DESC, pv DESC LIMIT 20;
```

## 第六步：运维操作

```sql
-- 暂停 Pipe（如需维护）
ALTER PIPE ods.kafka_user_events_pipe PAUSE;

-- 恢复 Pipe
ALTER PIPE ods.kafka_user_events_pipe RESUME;

-- 暂停 DWD 动态表
ALTER DYNAMIC TABLE dwd.user_events_clean SUSPEND;

-- 恢复 DWD 动态表（DWS 会跟随自动恢复触发）
ALTER DYNAMIC TABLE dwd.user_events_clean RESUME;

-- 排查刷新失败
SHOW DYNAMIC TABLE REFRESH HISTORY dwd.user_events_clean LIMIT 10;
```

## 架构说明

```
Kafka (kafka.prod.com:9092, topic: user_events, JSON)
        │
        ▼  [Pipe: kafka_user_events_pipe]
ods.user_events          ← 原始落地表，保留所有字段
        │
        ▼  [Dynamic Table, TARGET_LAG = '2 minutes']
dwd.user_events_clean    ← 过滤无效 event_type，新增 dt 分区字段
        │
        ▼  [Dynamic Table, TARGET_LAG = DOWNSTREAM]
dws.page_daily_stats     ← 按天按 page 聚合 UV / PV
```

关键设计决策：
- ODS 层用 Pipe + READ_KAFKA 持续摄入，无需调度，Kafka 消息自动落地。
- DWD 用动态表增量刷新，TARGET_LAG = '2 minutes' 控制延迟上限。
- DWS 用 TARGET_LAG = DOWNSTREAM，DWD 每次刷新完成后自动触发 DWS 更新，保持数据一致性。
- UV 使用 COUNT(DISTINCT user_id) 按天去重，PV 使用 COUNT(*) 统计总行数。
- event_type 白名单过滤写在 DWD 层，DWS 直接消费干净数据，职责清晰。
