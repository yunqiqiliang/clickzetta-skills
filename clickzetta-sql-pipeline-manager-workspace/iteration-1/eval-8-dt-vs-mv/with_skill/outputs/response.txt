## 动态表 vs 物化视图：核心区别

| 维度 | 动态表（Dynamic Table） | 物化视图（Materialized View） |
|---|---|---|
| 刷新方式 | 自动增量刷新，系统驱动 | 手动触发或定时全量刷新 |
| 刷新粒度 | 增量（只处理新增/变更数据） | 全量重算 |
| 延迟级别 | 秒/分钟级 | 分钟/小时/天级，按需配置 |
| 调度控制 | 由 TARGET_LAG 控制，系统自动执行 | REFRESH MANUAL 或 AUTO EVERY |
| 适用场景 | 实时/近实时 ETL、多层级联管道 | 固定聚合加速、BI 查询、低频刷新 |
| CDC 支持 | 支持（配合 Table Stream） | 不支持 |

---

## 你的场景：每天凌晨跑一次聚合报表

推荐使用**物化视图（Materialized View）**。

理由：
- 你的需求是"每天一次"，属于低频定时刷新，不需要实时增量能力
- 物化视图的 `REFRESH AUTO EVERY '1 days'` 正好匹配这个节奏
- 聚合报表是固定维度的预计算，物化视图的全量刷新语义更直观、更稳定
- 动态表的增量刷新优势（秒/分钟级、增量计算）在这个场景下完全用不上，反而引入不必要的复杂度

---

## 建议 SQL

```sql
-- 每天凌晨自动刷新的聚合报表物化视图
CREATE OR REPLACE MATERIALIZED VIEW dw.mv_daily_report
  COMMENT '每日聚合报表，凌晨自动刷新'
  REFRESH AUTO EVERY '1 days'
  VCLUSTER = default_ap
AS
SELECT
  DATE(created_at) AS day,
  region,
  COUNT(*) AS order_cnt,
  SUM(amount) AS total_revenue,
  COUNT(DISTINCT user_id) AS uv
FROM dwd.orders_clean
GROUP BY 1, 2;
```

如果你希望精确控制刷新时间点（比如严格在凌晨 2 点执行），可以改用 `REFRESH MANUAL`，然后通过外部调度器（如 cron）在指定时间触发：

```sql
-- 手动刷新模式
CREATE OR REPLACE MATERIALIZED VIEW dw.mv_daily_report
  COMMENT '每日聚合报表，手动触发刷新'
  REFRESH MANUAL
  VCLUSTER = default_ap
AS
SELECT
  DATE(created_at) AS day,
  region,
  COUNT(*) AS order_cnt,
  SUM(amount) AS total_revenue,
  COUNT(DISTINCT user_id) AS uv
FROM dwd.orders_clean
GROUP BY 1, 2;

-- 在调度任务中执行
REFRESH MATERIALIZED VIEW dw.mv_daily_report;
```

## 总结

- 实时/增量/多层 ETL → 动态表
- 定时聚合报表/BI 加速 → 物化视图
- 你的场景（每天一次聚合）→ 物化视图，用 `REFRESH AUTO EVERY '1 days'` 或 `REFRESH MANUAL` 配合外部调度
