要捕获 ods.orders 表的 UPDATE 和 DELETE 变更并同步到 dwd.orders_dim，需要使用 Table Stream + Dynamic Table 的组合方案。

## 方案说明

- UPDATE/DELETE 变更捕获 → 必须使用 MODE = STANDARD 的 Table Stream
- STANDARD 模式下，UPDATE 会产生两条记录：update_preimage（更新前）和 update_postimage（更新后）
- DELETE 会产生 _change_type = 'delete' 的记录
- 用 Dynamic Table 自动消费 Stream，执行 MERGE 写入目标表

## 完整 SQL

### Step 1：在 ods.orders 上创建 STANDARD 模式的 Table Stream

```sql
CREATE TABLE STREAM ods.orders_stream
  ON TABLE ods.orders
  MODE = STANDARD
  COMMENT '捕获 orders 表的 INSERT/UPDATE/DELETE 变更';
```

### Step 2：创建 Dynamic Table 消费 Stream，MERGE 到 dwd.orders_dim

```sql
CREATE OR REPLACE DYNAMIC TABLE dwd.orders_dim
  TARGET_LAG = '1 minutes'
  VCLUSTER = default_ap
  COMMENT 'CDC 同步自 ods.orders，含 UPDATE/DELETE 处理'
AS
MERGE INTO dwd.orders_dim AS target
USING (
  SELECT *
  FROM ods.orders_stream
  WHERE _change_type IN ('insert', 'update_postimage', 'delete')
) AS src
ON target.order_id = src.order_id
WHEN MATCHED AND src._change_type = 'delete'
  THEN DELETE
WHEN MATCHED AND src._change_type = 'update_postimage'
  THEN UPDATE SET
    target.user_id       = src.user_id,
    target.amount        = src.amount,
    target.status        = src.status,
    target.updated_at    = src._change_timestamp
WHEN NOT MATCHED AND src._change_type = 'insert'
  THEN INSERT (order_id, user_id, amount, status, created_at)
       VALUES (src.order_id, src.user_id, src.amount, src.status, src.created_at);
```

> 注意：如果 dwd.orders_dim 表还不存在，需要先建表，再创建上面的 Dynamic Table。

### （可选）如果 dwd.orders_dim 尚未创建

```sql
CREATE TABLE IF NOT EXISTS dwd.orders_dim (
  order_id   STRING,
  user_id    STRING,
  amount     DECIMAL(10, 2),
  status     STRING,
  created_at TIMESTAMP,
  updated_at TIMESTAMP
);
```

## 验证

```sql
-- 确认 Stream 已创建
SHOW TABLE STREAMS LIKE 'orders_stream';

-- 查看当前未消费的变更（可选，消费后数据会被标记已读）
SELECT _change_type, COUNT(*) FROM ods.orders_stream GROUP BY 1;

-- 确认 Dynamic Table 已创建并开始刷新
SHOW TABLES WHERE is_dynamic = true;

-- 查看刷新历史，确认无报错
SHOW DYNAMIC TABLE REFRESH HISTORY dwd.orders_dim LIMIT 5;
```

## 关键说明

| _change_type        | 含义                     | 处理方式         |
|---------------------|--------------------------|------------------|
| insert              | 新增行                   | INSERT 到目标表  |
| update_preimage     | UPDATE 前的旧值（忽略）  | 不处理           |
| update_postimage    | UPDATE 后的新值          | UPDATE 目标表    |
| delete              | 删除行                   | DELETE 目标表    |

- Stream 数据只能消费一次，Dynamic Table 会自动管理消费状态，无需手动干预
- 若长时间不消费（超出源表 data_retention_days），变更数据会丢失，建议保持 Dynamic Table 处于 RESUME 状态
- TARGET_LAG 可根据业务延迟要求调整，最低支持秒级（如 '30 seconds'）
